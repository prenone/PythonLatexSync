API Documentation

This API allows users to push and pull files with authentication. Below is a description of the available endpoints and how to use them.

1. Upload a File (Push)

POST /push/<user>/<write_password>/<filename>

Uploads a file to the server after verifying the user's write permissions.

Parameters

  • user: The username for authentication.
  • write_password: The password for write access.
  • filename: The name of the file to be saved on the server.

Request Example (Using curl)

curl -X POST -F "file=@/path/to/your/file.txt" http://127.0.0.1:5000/push/admin/write_secret/file.txt

2. Download a File (Pull)

GET /pull/<user>/<read_password>/<filename>

Downloads a file from the server after verifying the user's read permissions.

Parameters

  • user: The username for authentication.
  • read_password: The password for read access.
  • filename: The name of the file to be downloaded. Note that the file will always be downloaded as lol.txt.

Request Example (Using curl)

curl -O http://127.0.0.1:5000/pull/admin/read_secret/file.txt

Security and Headers

When downloading files using the GET /pull endpoint, caching is completely disabled to ensure the file is always fetched fresh from the server.
